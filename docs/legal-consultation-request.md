## 0. 자문 목적

P0a (RSS 워커 + DB 적재 + 결제 모듈) 진입 직전 시점에 한국 법무 위험을 사전 점검한다. 다음 5개 영역에서 (a) 위반 위험도, (b) 위반 시 처벌 수위, (c) 회피·완화 권고를 받고자 한다.

1. 자본시장법 — 투자 분석 카드 (Pro+ 전용)
2. 명예훼손·정통망법 — 매체별 보도 표시 + AI 요약 + Trust Tag
3. 전자상거래법·표시광고법 — 제휴 커머스 (쿠팡파트너스/11번가)
4. 게임산업법·청소년보호법 — 코인 시스템 + 결제 + 자동전환 동의 (2025-02-14 시행)
5. 개인정보보호법 (PIPA) — 대기자 폼 + 분석 이벤트 (PostHog)

각 영역의 결정 근거는 첨부 ADR (저장소 `docs/adr/`)에 상세히 있다. 필요 시 GitHub 저장소 (https://github.com/sgeniusk/tteuniyu) 열람 권한 제공 가능.

## 1. 자본시장법 — 투자 분석 카드

### 1.1 사실관계

ADR-015 §2.2에 따라 Pro/Creator/Leader 구독자에게 다음을 표시할 계획이다.

- **무료 사용자** — 클러스터에 `category=investment` 자동 분류 시 노란 배지 "💰 투자 정보" + 카드 하단 고지 "⚠ 자본시장법상 투자 자문이 아닙니다. 신중히 판단하세요."
- **Pro+ 사용자** — `<InvestmentImpactCard>` 표시
  - 관련 종목·자산명 (텍스트만, 종목코드 005930 OK, "거래" 단어 X)
  - 외신 비교 (Bloomberg/Reuters/FT 보도 요약, 출처 링크)
  - 공시·발표 요약 (DART/SEC 등 1차 자료 LLM 요약)
  - 분석가 의견 요약 (공개 보도에 등장한 분석가 코멘트 LLM 요약)

가격·시세·차트·매수/매도 추천 일체 X (V0.5+ 별도 ADR-016 발의 후 검토).

LLM output validator로 "예측", "추천", "전망", "매수", "매도", "목표가" 단어 0건 강제.

### 1.2 질의 사항

- **Q1.1** Pro+ 사용자에게 "공개 보도 + 공시 요약" 형태로 분석가 의견을 요약 제공하는 행위가 자본시장법 §6 "유사투자자문업" 등록 요건에 해당하는가
- **Q1.2** 회피하려면 추가로 어떤 단어/표현을 LLM 출력에서 차단해야 하는가 (현재 "예측·추천·전망·매수·매도·목표가" 외)
- **Q1.3** 종목코드(005930) 노출은 안전한가 (vs 종목명 "삼성전자" 단독 노출과 차이)
- **Q1.4** 자본시장법 §178 시세조작 정보 유포 회피를 위해 LLM 워커가 출력한 모든 분석에 prompt_version + model + 출처 + 작성일시 메타데이터를 카드 하단에 표시할 계획이다. 이로 충분한가
- **Q1.5** Pro 가입자가 자기 분석을 SNS에 공유할 때 "본 분석은 뜬이유 AI 요약" 출처 표기를 강제해야 하는가, 아니면 권장만으로 충분한가

## 2. 명예훼손·정통망법 — 매체 표시 + AI 요약 + Trust Tag

### 2.1 사실관계

ADR-009 Amendment 1 + ADR-015 §2.1·2.3에 따라 다음 구조로 매체 정보를 표시한다.

- **매체별 보도 list** — 매체명 + 발행 시간 + 헤드라인 + 회색 dot (slate-400 단일). 4분류 라벨 (보수/진보/중도/공영) 일체 X. 출처 링크 명시
- **Trust Tag 3종** — 클러스터 단위로 부여 (개별 매체 X)
  - `hoax` (빨강) — `hoax_likelihood ≥ 0.7` (LLM 평가)
  - `clickbait` (빨강) — `clickbait_score ≥ 0.7` (LLM 평가, 본론 추출과 결합)
  - `low_confidence` (노랑) — `sample_size < 5` OR `single_source_only`
- **AI 요약** — 매체별 헤드라인 + 본론 LLM 요약 (≤ 15% copy ratio, harness 강제)
- **정정 요청 폼** — 매체가 1-click으로 정정 요청 가능 (P0a UX). 1주 ≥ 3건 누적 시 자동 태그 제거 + 사람 검토 큐

LLM 모델 (Phase 1 Gemini 2.0 Flash → Phase 2 맥미니 → Phase 3 GPU)과 prompt_version (`trust_signal_v1`), 임계값(0.7) 모두 GitHub에 공개.

### 2.2 질의 사항

- **Q2.1** Trust Tag `hoax`/`clickbait`를 클러스터 단위로 부여하지만 그 안에 특정 매체의 보도가 포함된 경우, 해당 매체에 대한 명예훼손(형법 §307 또는 정통망법 §70) 위험이 있는가
- **Q2.2** "AI 자동 분석 결과입니다. 매체에 정정 요청 가능합니다" 고지 + 1-click 정정 요청 + 1주 ≥ 3건 자동 태그 제거 — 이 면책 구조로 충분한가
- **Q2.3** AI 요약이 원문과 의미를 다르게 전달할 경우 정통망법 §44-2 (정정 요청권) 외에 추가로 어떤 권리가 매체에 있는가
- **Q2.4** 매체 dot을 회색 단일로 통일하고 4분류 라벨을 폐기한 결정이 명예훼손·허위사실 유포 위험을 충분히 회피하는가
- **Q2.5** AI 요약·Trust Tag로 인한 분쟁 발생 시 자체 모니터링 의무 (정통망법 §44-3)가 있는가, 사용자 신고 의존이 가능한가
- **Q2.6** 외신(Reuters/AP/BBC/NYT/Bloomberg/Nikkei) 헤드라인·본론 요약 시 추가로 고려할 국제 저작권 이슈가 있는가 (한미·한일 저작권 협정)

## 3. 전자상거래법·표시광고법 — 제휴 커머스 (쿠팡파트너스·11번가)

### 3.1 사실관계

ADR-007에 따라 다음 원칙을 적용한다.

- **저장 금지** — 쿠팡/11번가/아마존 상품 데이터(title/image/price/stock/reviews/description)를 어떤 DB 테이블에도 저장 X. 런타임 API 호출만, no-cache, no log of raw payload
- **수동 큐레이션** — `config/affiliate_manual_curation.yaml`에 1일 1회 운영자 수동 등록만 허용
- **분리 영역** — 실시간 이슈 카드(`/widget`) 영역에만 노출. Coverage 분석 페이지(`/cluster/[id]`, `/methodology`, `/dispute`, `/outlet-compare`)에는 절대 X (ADR-005 P12 격리)
- **카테고리 제외** — 정치·선거·사고·재해·의료 카테고리 클러스터에는 자동 매칭 X
- **표시 문구** — `<AffiliateCard>` 하단에 "이 카드는 제휴 링크입니다 (쿠팡 파트너스)" 표기

### 3.2 질의 사항

- **Q3.1** "이 카드는 제휴 링크입니다 (쿠팡 파트너스)" 단일 표기로 표시광고법 §3 (표시 의무)·전자상거래법 §13 (정보 제공 의무) 충족 여부
- **Q3.2** "이 포스팅은 [쿠팡 파트너스] 활동의 일환으로, 일정액의 수수료를 제공받을 수 있습니다" 같은 표준 문구가 추가로 필요한가
- **Q3.3** 클러스터 맥락(예 "전세 사기" 이슈)과 매칭된 상품(예 "자취 가전")이 신뢰 훼손·소비자 오인 가능성이 있다는 우려 — P0w에서는 1일 1회 수동 큐레이션만 허용하고 자동 매칭은 V0.5+로 보류한 결정으로 충분한가
- **Q3.4** 쿠팡파트너스 ToS의 "제휴 링크 변형 금지" 조항 — 우리 시스템이 운영자 수동 등록한 affiliate_url에 자체 redirect를 거치지 않고 그대로 사용하는데 추가 검토 필요한가
- **Q3.5** "정치·선거·사고·재해·의료" 카테고리 자동 제외 외에 추가로 제외해야 할 민감 카테고리가 있는가 (예 종교, 부동산, 금융 상품)

## 4. 게임산업법·청소년보호법·전자상거래법 — 코인 + 구독 + 자동전환

### 4.1 사실관계

ADR-011에 따라 Hybrid Coin + 3-tier Subscription 모델을 적용한다.

- **코인 충전** — Free 사용자가 결제로 코인 충전 (예 ₩1,000 → 100 coins). 광고 시청·앱 열기로 무료 획득 가능 (1일 1회 1 coin / 광고 클릭 2 coin / 광고 끝까지 시청 10 coin)
- **코인 사용처** — Pro+ 분석 카드 1회 열람, OG 카드 다운로드, 키워드 알림 추가 등
- **3 Tier 구독** — Pro $4.99/월, Creator $9.99/월, Leader $19.99/월
- **자동전환** — 무료 체험 X (Trial 없음, 코인이 effective free trial 역할). 자동 결제 갱신 시 결제 7일 전 알림 (2025-02-14 시행 자동전환 동의 의무 준수)
- **잔여 환불** — 유료 코인 잔액 사용자 요청 시 비례 환불 (전자상거래법 §17 청약철회권 준수)
- **연령 제한** — 만 14세 이상 가입 가능, 만 19세 미만은 코인 결제·구독 결제 시 법정대리인 동의 절차 (PG 제공)

### 4.2 질의 사항

- **Q4.1** 코인 시스템이 게임산업법 §2 "게임물" 정의에 해당할 가능성 — 광고 시청 → 코인 획득 → Pro 분석 열람 구조가 "사행성" 또는 "게임성"으로 해석될 위험
- **Q4.2** 광고 시청 보상 코인이 청소년보호법 상 사행행위 또는 사행성 게임물에 해당하는가 (만 19세 미만 보상 한도가 있는가)
- **Q4.3** 자동전환 동의 (2025-02-14 시행)의 구체 의무 이행 방법 — 결제 7일 전 알림 채널 (이메일/SMS/푸시 중 어느 것이 법적으로 안전한가)
- **Q4.4** 잔여 코인 환불 — 유료 코인과 무료 코인을 혼용 사용한 경우 어느 것을 먼저 차감하는 것이 전자상거래법 §17 청약철회권에 부합하는가
- **Q4.5** Trial 없는 즉시 결제 모델 — Apple App Store / Google Play Store 결제 모듈 사용 시 추가 고려 사항 (P0b 결제 모듈 도입 단계)
- **Q4.6** Creator $9.99 → Leader $19.99 업그레이드 시 차액 일할 환불 vs 다음 사이클부터 적용 — 전자상거래법 §17 정합 방식

## 5. 개인정보보호법 (PIPA)·정통망법 — 폼 + 분석 이벤트

### 5.1 사실관계

P0a부터 다음 데이터를 수집한다.

- **대기자 폼** — Pro/Creator/B2B 3종 (이메일 + 도메인/회사명/사용 케이스 + 4점 의향 점수 등)
- **분석 이벤트** — PostHog Cloud (us.posthog.com, project `tteuniyu-prod`). IP, User-Agent, 페이지 이동, 클릭 이벤트 11종 자동 수집
- **저장 위치** — Supabase (서울 리전) + PostHog (US 리전)
- **로그 보존 기간** — 13개월 (정통망법 권고)

### 5.2 질의 사항

- **Q5.1** PostHog (US 리전) 사용 시 PIPA §28-8 국외 이전 동의 절차 — 폼 제출 시점에 동의 받아야 하는 항목 목록
- **Q5.2** 대기자 이메일 수집 시 필수 동의 항목 — 수집 목적 / 보유 기간 / 동의 거부권 명시 — 표준 문구 권고
- **Q5.3** 폼 제출자가 미성년자 (만 14세 미만)인 경우 추가 절차
- **Q5.4** PostHog 이벤트에 IP·UA가 자동 포함됨 — 익명 분석으로 분류되는 한도 (PIPA 가이드라인)
- **Q5.5** 13개월 보존 후 자동 파기 절차 — 시스템 자동 cron 외에 추가 의무 (예 분기 1회 점검 보고서)
