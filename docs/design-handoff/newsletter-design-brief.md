# 뜬이유 뉴스레터 디자인 브리프 — Daily / Weekly

> 최종 갱신 2026-05-19
> 목적 — 이메일 뉴스레터(Daily Digest·Weekly Digest)의 디자인 개편 명세.
> 사용처 — 본 문서 하단 "Claude 디자인 프롬프트"를 Claude에 붙여 HTML 시안 생성.
> 구현 연결 — 확정 시안은 `apps/worker/src/tteuniyu_worker/digest_template.py`에 반영.

---

## 1. 배경

뜬이유 Daily Digest는 ADR-017 기반 이메일 뉴스레터다. 현재 `digest_template.py`가
HTML을 렌더하나 디자인이 기초 수준이다. baeksang.dev 일일 뉴스레터의 구조를
참고해 개편한다.

### baeksang.dev에서 가져오는 것
- 에디토리얼 도입글 — 다이제스트 첫머리에 그날의 한 문단
- 피라미드식 항목 구성 — 제목 → 요약 → 상세 순
- 푸터 — 공유·아카이브·이전/다음 호 네비게이션

### baeksang.dev에서 가져오지 않는 것
- "지금 할 일" 행동 지침 — 뜬이유는 이슈 *분석* 제품이지 행동 유도 제품이 아니다
- 출처별 분류(GitHub/ArXiv 등) — 뜬이유는 cluster 단위. 분류는 velocity 순위 + 카테고리

### 뜬이유 뉴스레터의 핵심 원칙
독자에게 무엇을 하라고 시키지 않는다. **"왜 이 이슈가 떴고, 매체별로 어떻게
다뤘는가"**를 전달한다. 이것이 실검·일반 뉴스레터와 뜬이유를 가르는 지점이다.

## 2. 데이터 모델 (디자인이 표현할 실제 필드)

`digest_template.py`의 dataclass 기준. 디자인은 이 필드만 사용할 수 있다.

### 다이제스트 전체 (`DigestPayload`)
- `digest_date` — 발행일
- `clusters[]` — 이슈 카드 목록
- `custom_topic_matches[]` — 구독자가 등록한 키워드에 매칭된 이슈

### 이슈 카드 (`ClusterSummaryCard`)
| 필드 | 내용 | 디자인 표현 |
|---|---|---|
| `title` | 이슈 대표 제목 (≤60자) | 카드 헤드라인 |
| `category` | politics/society/economy/international/tech_science/culture_sports/lifestyle | 카테고리 아이콘+라벨 |
| `outlets_count` | 보도 매체 수 | "12개 매체 보도" |
| `trust_tag` | hoax/clickbait/low_confidence/investment 또는 없음 | Trust Tag 배지 (아래 §5) |
| `why_trending` | 왜 떴는지 1~2문장 (≤500자, 실제론 1문장 권장) | 카드 본문 |

### 커스텀 토픽 매칭 (`CustomTopicMatch`)
- `keyword` — 구독자가 등록한 키워드
- `cluster_title` — 매칭된 이슈 제목
- `outlets_count` — 보도 매체 수

### 시안 작성 시 추가 제안 가능 (현재 모델에 없음 — 채택 시 백엔드 확장)
- `latest_article` — 이슈의 최신 전개 헤드라인 + 시각 ("23분 전 · ○○")
- 에디토리얼 도입글 — LLM 생성 1문단 (비용 발생, 채택 시 별도 결정)

## 3. Daily Digest 구조

```
┌─────────────────────────────────────┐
│ [헤더]                               │
│   📬 뜬이유 Daily · 2026-05-19 (월)   │
│   에디토리얼 — "어제 한국은 ○○ 이슈가  │
│   가장 크게 다뤄졌습니다" (1문단)      │
├─────────────────────────────────────┤
│ [지금 뜨는 이슈]  ← velocity 순위 5~7개│
│                                      │
│   01  이슈 제목                       │
│       why_trending 1문장              │
│       🏛 정치 · 12개 매체 · [배지]     │
│       → 매체별 보도 비교 보기 (링크)   │
│                                      │
│   02  ...                            │
├─────────────────────────────────────┤
│ [내 관심 토픽]  ← custom_topic_matches │
│   "전세사기" → 매칭 이슈 2건           │
├─────────────────────────────────────┤
│ [푸터]                               │
│   본 정보는 투자 자문이 아닙니다 (고지) │
│   공유 · 1-click 수신거부 · 지난 호    │
└─────────────────────────────────────┘
```

- 이슈는 **velocity 순위** (최근 활발히 보도된 순). 카테고리 혼합.
- 카드당 텍스트는 baeksang(250~350자)보다 가볍게 — 제목 + why_trending 1문장 + 메타.
- "매체별 보도 비교 보기" 링크 → `https://tteuniyu.vercel.app/cluster/{cluster_id}`

## 4. Weekly Digest 구조

Daily와 같은 시각 언어, 콘텐츠 범위만 다르다.

```
┌─────────────────────────────────────┐
│ 📬 뜬이유 Weekly · 5/12~5/18          │
│   에디토리얼 — "이번 주 한국을 움직인  │
│   흐름" (2~3문단)                     │
├─────────────────────────────────────┤
│ [이번 주 가장 큰 이슈 Top 5]          │
│   주간 누적 보도량 기준               │
│   각 이슈에 "주중 어떻게 전개됐나" 한 줄│
├─────────────────────────────────────┤
│ [이번 주 카테고리 흐름]               │
│   정치 N건 · 경제 N건 ... 비중 표시   │
├─────────────────────────────────────┤
│ [푸터] — Daily와 동일                 │
└─────────────────────────────────────┘
```

- Daily가 "어제 24시간"이면 Weekly는 "지난 7일".
- Weekly Top은 velocity가 아닌 **주간 누적 보도량**(outlets_count 합 또는 기사 수).

## 5. 비협상 제약 (디자인이 반드시 지킬 것)

- **1-click 수신거부** — 푸터에 수신거부 링크 필수 (정통망법 §50). 눈에 띄게.
- **자본시장법 고지** — 푸터에 "본 정보는 투자 자문이 아닙니다" 문구 필수 (변호사 §1.4).
- **Trust Tag 라벨** — 고정 문구·색 (변호사 §1.3, ADR-015 Amendment 2).
  - `hoax` → "⚠ 검증 필요" (#dc2626 빨강)
  - `clickbait` → "⚠ 제목-본문 괴리 가능성" (#dc2626 빨강)
  - `low_confidence` → "⚠ 표본 부족" (#f59e0b 주황)
  - `investment` → "🏢 기업 관련 이슈" (#f59e0b 주황)
- **금지 표현** — "실시간 검색어/실검/인기 검색어" 등 사용 금지 (CLAUDE.md Naming Ban). "실시간 이슈/지금 뜨는 이슈"는 허용.
- **이미지 없음** — 뜬이유 P0는 이미지 미저장(rule 3). 뉴스레터도 텍스트·이모지·도형만. 사진/썸네일 X.

## 6. 이메일 HTML 기술 제약

이건 웹페이지가 아니라 **이메일**이다. Gmail·Naver Mail·Outlook에서 깨지지 않아야 한다.

- **inline CSS만** — `<style>` 블록·외부 stylesheet 금지 (많은 클라이언트가 무시/제거)
- **테이블 레이아웃** — `flexbox`·`grid` 금지. `<table>` 기반 레이아웃
- **최대 너비 600px** — 본문 컨테이너
- **웹폰트 금지** — Pretendard 등 로드 안 됨. `font-family`에 시스템 폰트 fallback (`-apple-system, 'Malgun Gothic', sans-serif`)
- **다크 배경 주의** — 뜬이유 웹은 다크(slate-950)이나, 이메일 다크 배경은 클라이언트별 반전 처리가 제각각. **밝은 배경(흰색/연회색) 권장**, 액센트만 teal
- **이모지는 OK** — 카테고리 아이콘 등에 활용 가능 (이메일에서 안정적으로 렌더)

### 브랜드 색
- 액센트 — teal `#14b8a6`
- 텍스트 — 거의 검정 `#1e293b`, 보조 회색 `#64748b`
- 경고 — 빨강 `#dc2626`, 주황 `#f59e0b` (Trust Tag 전용)

## 7. Claude 디자인 프롬프트

아래 블록을 그대로 복사해 Claude에 붙여넣으면 HTML 이메일 시안을 받을 수 있다.
Daily를 먼저 받고, 확정 후 같은 톤으로 Weekly를 요청한다.

```
한국 뉴스 이슈 분석 서비스 "뜬이유"의 Daily Digest 이메일 뉴스레터 HTML 시안을
만들어줘. 단일 HTML 파일로, 실제 이메일 클라이언트(Gmail·Naver Mail)에서
깨지지 않게.

[제품 성격]
- 뜬이유는 "한국 언론이 지금 무엇을 다루는가 + 왜 떴는가 + 매체별로 어떻게
  다뤘는가"를 보여주는 이슈 분석 서비스다.
- 실검(실시간 검색어)이 아니다. 속보 경쟁이 아니라 "맥락"을 판다.
- 독자에게 행동을 지시하지 않는다. 이해를 돕는다.

[이메일 기술 제약 — 반드시 준수]
- inline CSS만. <style> 블록·외부 CSS 금지.
- <table> 기반 레이아웃. flexbox·grid 금지.
- 본문 최대 너비 600px, 가운데 정렬.
- 웹폰트 금지. font-family: -apple-system, 'Malgun Gothic', sans-serif.
- 밝은 배경(흰색/연회색). 액센트 색만 teal #14b8a6.
- 이미지·사진 없음. 텍스트·이모지·CSS 도형만.

[구조]
1. 헤더 — "📬 뜬이유 Daily · 2026-05-19 (월)" + 에디토리얼 1문단
   (예: "어제 한국은 삼성전자 노사 협상이 가장 크게 다뤄졌습니다. ...")
2. "지금 뜨는 이슈" 섹션 — 이슈 카드 6개. 각 카드:
   - 순위 번호 (01~06)
   - 이슈 제목 (굵게)
   - 왜 떴는지 1문장
   - 메타 줄 — 카테고리(🏛 정치 등) · "12개 매체 보도" · Trust Tag 배지(해당 시)
   - "매체별 보도 비교 보기 →" 링크
3. "내 관심 토픽" 섹션 — 키워드별 매칭 이슈 (예: "전세사기 → 매칭 2건")
4. 푸터:
   - "본 정보는 투자 자문이 아닙니다." 고지 문구
   - 공유 / 1-click 수신거부 / 지난 호 보기 링크

[Trust Tag 배지 — 고정 문구·색]
- "⚠ 검증 필요" (빨강 #dc2626)
- "⚠ 제목-본문 괴리 가능성" (빨강 #dc2626)
- "⚠ 표본 부족" (주황 #f59e0b)
- "🏢 기업 관련 이슈" (주황 #f59e0b)

[샘플 데이터 — 이걸로 채워줘]
- 발행일: 2026-05-19 (월)
- 에디토리얼: 어제 한국은 삼성전자 노사 협상 임박 이슈가 가장 크게 다뤄졌습니다.
- 이슈 카드 6개:
  01 "삼성전자 파업 임박, 중노위 긴급조정권 시사" / 왜: 노사 협상 결렬 후
     정부가 개입 신호를 보내며 14개 매체가 일제히 보도. / 정치 · 14개 매체 · (배지 없음)
  02 "코스피 7443 마감, 외국인 순매도 지속" / 왜: 국채금리 상승과 셀코리아
     우려로 11개 매체가 동시 분석. / 경제 · 11개 매체 · 🏢 기업 관련 이슈
  03 "이재명 대통령, 트럼프와 통화" / 왜: 한미 정상 통화 내용이 공개되며
     외교 라인 전반이 보도. / 국제 · 9개 매체 · (배지 없음)
  04 "김정은, 전군 지휘관 소집" / 왜: 북한 군사 동향에 대한 분석 보도가
     집중. / 국제 · 8개 매체 · ⚠ 표본 부족
  05 "전세사기 피해 구제 특별법 후속 논의" / 왜: 피해자 단체 기자회견 후
     후속 보도. / 사회 · 7개 매체 · (배지 없음)
  06 "국민연금 개편안 윤곽" / 왜: 보험료율 조정 방향이 보도되며 관심 집중.
     / 경제 · 6개 매체 · (배지 없음)
- 관심 토픽 매칭: "전세사기" → 1건, "반도체" → 2건

[톤]
차분하고 신뢰감 있게. 과한 장식 X. 정보 위계가 명확하게 — 제목이 먼저
눈에 들어오고, 왜 떴는지가 그 다음, 메타 정보는 작게.

완성된 HTML을 artifact로 보여줘. 이후 같은 디자인 언어로 Weekly Digest
버전도 요청할 거야.
```
