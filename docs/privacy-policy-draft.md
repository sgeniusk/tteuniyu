## 0. 적용 범위

뜬이유 (TTEUN-IYU, 이하 "회사")는 본 개인정보 처리방침을 통해 회사가 운영하는 다음 서비스에서 이용자의 개인정보를 어떻게 수집·이용·보관·파기하는지 안내한다.

- 웹사이트 — https://tteuniyu.com (예정)
- 위젯 임베드 — `/embed/iframe`, `/widget.js`
- 베타 대기자 폼 — `/preorder/pro`, `/preorder/creator`, `/inquiries/b2b`
- (V0.5+) iOS / Android 모바일 위젯
- (P0b+) 결제 모듈

본 문서는 **이용자가 본인의 개인정보가 어떻게 다뤄지는지 명확히 알 수 있도록** 작성한다.

| 구분 | 항목 | 수집 방법 |
|---|---|---|
| **자동** | IP 주소 | PostHog SDK |

| 전자상거래법 §6 (소비자 불만·분쟁 처리 기록) | 3년 |
| 통신비밀보호법 §15-2 (서비스 방문 기록) | 3개월 |
| 정정 요청·삭제 요청 처리 기록 | 3년 |

### 3.2 파기 절차

- **자동 파기 cron** — 매일 03:00 KST 실행
  - `withdrawn_at + 30일` 경과 사용자 자동 파기 (동의 철회)
  - `required_consent_at + 13개월` 경과 분석 이벤트 자동 파기
  - 파기 결과는 `audit_logs.retention_cron` 테이블에 기록 (실행일·대상 수·완료 수·실패 건수)
- **수동 파기** — 이용자 요청 시 30일 이내 파기 (이메일로 처리 결과 통지)
- **파기 방법**
  - 전자 파일 — 데이터베이스에서 SQL DELETE + 백업에서도 즉시 삭제
  - 종이 출력물 — 분쇄 또는 소각 (P0 단계에서는 종이 출력물 없음)
- **PostHog 파기** — PostHog Inc. 측에 13개월 retention setting 설정 + 분기 1회 점검

| 항목 | 내용 |
|---|---|
| **이전받는 자** | PostHog Inc. (Delaware, USA) — privacy@posthog.com |
| **이전 국가** | 미국 |
| **이전 시점** | 서비스 이용 및 이벤트 발생 시 (실시간) |
| **이전 방법** | HTTPS API 호출 (TLS 1.3 이상) |
| **이전 항목** | IP 주소·User-Agent·페이지 URL·클릭 이벤트·브라우저/기기 정보 |
| **이전 목적** | 제품 분석·오류 분석·서비스 개선 |
| **보유 기간** | 13개월 또는 목적 달성 시까지 |

이용자가 PostHog 국외 이전을 거부한 경우 client에서 `posthog.opt_out_capturing()` 호출 + cookie 설정으로 이전 차단을 보장한다.

## 7. 만 14세 미만 아동 (PIPA §22-2)

회사는 만 14세 미만 아동의 개인정보를 원칙적으로 수집하지 않는다.

- 가입·대기자 등록 시 생년 입력을 통해 만 14세 미만 자동 거부
- 거짓 정보 입력으로 가입 후 적발 시 즉시 계정 차단 + 데이터 파기 + 보호자 통지
- 만 14세 ~ 19세 미만 — P0b 결제 화면에서 법정대리인 동의 절차 (PG 제공)

## 8. 정정 요청 (정통망법 §44-2)

본 서비스의 AI 요약·Trust Tag (검증 필요·제목-본문 괴리 가능성·표본 부족·기업 관련 이슈)에 의해 매체 또는 당사자가 권리 침해를 받았다고 판단할 경우 다음 절차로 정정 요청을 할 수 있다.

| 단계 | 절차 |
|---|---|
| 1. 1-click 정정 요청 | `/cluster/[id]` 디테일 페이지의 "정정 요청" 버튼 → 폼 제출 |
| 2. 분류 | 일반 정정 — 24시간 내 검토 큐 등록 / 매체·당사자 직접 — 태그 임시 숨김 검토 / 권리침해 소명 — 지체 없이 삭제·수정·임시조치 |
| 3. 결과 통지 | 처리 결과를 이메일로 통지 (영업일 기준 7일) |
| 4. 1주 ≥ 3건 누적 | 자동 태그 제거 + 사람 검토 큐 (변호사 권고 §2.5) |

회사는 정정 요청·수정 결정·삭제 결정의 모든 이력을 audit log에 보관한다 (3년).

## 9. 보안 조치

회사는 이용자의 개인정보 보호를 위해 다음 기술적·관리적 조치를 시행한다.

### 9.1 기술적 조치

- HTTPS 전구간 암호화 (TLS 1.3 이상)
- DB 암호화 (Supabase 기본 암호화 + at-rest)
- API 인증 (service_role 분리, NEXT_PUBLIC 키는 client 노출용으로만)
- PostHog autocapture 비활성화 + property_blacklist 강제
- 모든 LLM 호출에 prompt_version·model·input_hash·output_hash 로그
- harness 21종 자동 검증 (CI 통과 강제)

### 9.2 관리적 조치

- 개인정보 처리 직원 최소화 (현재 — Founder 1명)
- 분기 1회 privacy checklist 점검
- 사고 발생 시 72시간 이내 KISA 신고 (정통망법 §32-2)
- 이용자 정정 요청은 audit log에 기록
