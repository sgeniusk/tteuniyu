# 뜬이유 — 서비스 런칭 계획

> 최종 갱신 2026-05-16
> 목적 — "런칭까지 무엇을, 어떤 순서로" 하나의 문서로 본다.
> 과거 작업 상세는 git PR 히스토리·ADR에 있으므로 여기서는 다루지 않는다.

---

## 런칭의 정의

뜬이유의 "런칭"은 다음 상태를 말한다.

> 누구나 웹으로 접속해 오늘의 실시간 이슈를 신뢰할 수 있는 형태로 보고,
> 유료 전환 의향을 남길 수 있는 공개 서비스.

iOS 네이티브 위젯·결제 시스템은 런칭 *이후*다. 런칭은 "공개 웹 서비스 + 지불 의향 검증"이 기준이다.

---

## 지금 어디까지 왔나

데이터 파이프라인이 자동으로 돈다 — RSS 21개 매체 수집 → 클러스터링 → AI 요약 → 위젯 순위. 백엔드는 실데이터이고, cluster 상세의 일부 시각화(추이 그래프·보도 분포)만 아직 mock이다. 웹은 로컬에서만 뜨고, 아직 공개 배포되지 않았다.

한 줄 요약 — **엔진은 돌고 있고, 계기판 일부와 외장이 남았다.**

---

## 마일스톤

런칭까지 4개 단계. 각 단계는 "완료 기준"을 통과해야 다음으로 넘어간다.

### M1 — 신뢰 데모 완성

cluster 상세 페이지에 가짜가 하나도 없는 상태를 만든다. 뜬이유의 핵심 가치는 "신뢰"이므로 mock 데이터로 공개할 수 없다.

할 일.
- 추이 그래프 실데이터 — ADR-008 keyword_trends 워커 (네이버 데이터랩 무료 API 또는 자체 언급량 집계)
- 보도 분포 막대 실데이터 — 매체 성향 집계 (ADR-009 — 매체에 라벨 저장 금지, 런타임 집계만)
- summary staleness 해소 — 클러스터에 기사가 추가되면 요약 자동 갱신
- velocity 기반 순위 — "지금 급상승"을 시간당 기사 증가율로 계산

완료 기준 — `/cluster/[id]` 어느 페이지를 열어도 mock 표시가 없다.

### M2 — 공개 배포 준비

서비스를 인터넷에 올리고, 처음 방문자가 거부감 없이 쓸 수 있게 다듬는다.

할 일.
- Vercel 프로덕션 배포 + 도메인 연결
- 랜딩·카피 정리 — 첫 화면에서 "이게 뭔지" 5초 안에 전달
- 법무 페이지 — 개인정보처리방침·이용약관 게시 (초안 `docs/privacy-policy-draft.md` 존재)
- 분석 이벤트 — PostHog 연동, 핵심 행동 11종 추적 (`apps/web/lib/analytics/events.ts`)
- harness 18종 실구현 — 광고 분리·네이밍 금지 등 비협상 제약 자동 검증

완료 기준 — 외부인이 URL만으로 접속해 위젯을 보고, 개인정보 처리 안내를 확인할 수 있다.

### M3 — 소프트 런칭 ← 여기가 런칭

서비스를 공개하고, 초기 사용자를 받고, 지불 의향을 측정한다.

할 일.
- 공개 — 커뮤니티·SNS에 제한적 노출, 초기 유입 확보
- 대기자 명단 — 유료 기능 관심 등록 받기
- 지불 의향 측정 — PostHog로 Paid Intent Rate 추적
- Daily Digest 실발송 — Resend 도메인 verification 후 활성화
- 피드백 수집 채널 — 사용자 의견·정정 요청 창구

완료 기준 (= P0w Exit 게이트) — **Paid Intent Rate ≥ 4% AND 대기자 ≥ 100명**. 이 수치가 iOS 네이티브 위젯 착수 잠금을 푼다 (CLAUDE.md 14조, CI가 강제).

### M4 — 수익화 본격 (런칭 이후)

소프트 런칭에서 지불 의향이 확인되면 결제와 네이티브 앱으로 확장한다.

할 일.
- 결제 — 코인 + 3-tier 구독 (ADR-011)
- iOS Small Widget — P0w Exit 통과 시에만 Swift 착수 (ADR-006)
- Affiliate Commerce / Contextual Ads — 분리 영역 원칙 (ADR-005/007)
- Custom Topic 매칭 실가동 (ADR-018)
- Android·macOS 위젯 — 그 이후

## 우선순위 — 다음에 손댈 것

```
1순위  M1 추이 그래프 실데이터        ← cluster 상세의 가장 큰 mock
2순위  M1 summary staleness 해소     ← 데이터 정확성
3순위  M1 보도 분포 실데이터
4순위  M2 Vercel 배포 + 도메인
5순위  M2 법무 페이지 게시
```

M1을 끝내야 "신뢰할 수 있는 데모"가 되고, 그래야 M2 공개가 의미 있다. M1 → M2 → M3 순서는 바꾸지 않는다.
